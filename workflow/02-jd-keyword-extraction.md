# Step 2 — JD keyword extraction

Extract the top 15–20 ATS keywords and phrases from the JD.

## Categorise as:

- **Technical / domain skills** — specific tools, platforms, methodologies, certifications. Concrete things that are either present in a CV or not (e.g., "Salesforce CRM," "SQL," "ISO 9001," "Kubernetes").

- **Soft skills / competencies** — what the hiring team actually needs, read past the jargon. "Cross-functional stakeholder management" is real. "Team player" is noise — skip generic filler and only extract competencies that are clearly load-bearing for this role.

- **Title / seniority signals** — the level of person they're actually hiring for. Read between the lines: if the title says "Manager" but the requirements say "you'll be the only person doing X," they want a hands-on operator. Note both what's stated and what's implied.

- **Industry / sector terms** — domain knowledge the role requires (e.g., "financial services," "discrete manufacturing," "developer tooling"). These affect ATS ranking even when they're not "skills."

## How to use this list

Record the keywords internally. They'll be used in:
- Step 3 (gap analysis): checking which are in the profile
- Step 4 (bullet rewrite): weaving matching ones into bullets where accurate
- Step 5 (profile rewrite): including 2–3 most relevant in the profile paragraph

Don't report the keyword list to the user unless they ask. This is an internal extraction step. Proceed to step 3.
